Maak een quiz over een onderwerp naar keuze die je medestudenten kunnen spelen


  • download hier de template voor de quiz

Generieke requirements

  • Een eenmaal gegeven antwoord mag achteraf niet worden gewijzigd

  • De quiz moet uit precies 5 vragen bestaan

  • De quiz bestaat uit 1 HTML-pagina (Single Page Application) (CSS en Javascript staan wel in hun eigen bestand)


Algemene regels:



  • De code van elke deelopdracht is gereviewd door een medecursist

  • Elke deelopdracht is getest door een medecursist

  • Voor alle deelopdrachten is een testcase opgenomen in de index.html van de documentation map (vanaf US04)

  • Er is geen gebruik gemaakt van HTML-, JavaScript- en CSS-frameworks -libraries en -templates

  • De applicatie bevat geen alert- en confirm-boxes voor gebruikersinteractie. Alerts mogen wel gebruikt worden voor het weergeven van fouten.

  • De code bevat geen errors volgens de onderstaande validators

    • Validator HTML

    • Validator CSS

    • Validator JavaScript

  • De quiz functioneert correct op de laatste versie van Firefox en Google Chrome

  • De JavaScript-code bevat geen errors in de browserconsole

  • In de HTML-code staat geen JavaScript en CSS/styling

  • Alle code (HTML, CSS en JavaScript) moet correct ingesprongen zijn (Shift + Alt + F)

Als speler wil ik informatie zien over de quiz die ik ga spelen zodat ik weet hoe ik de quiz moet spelen en hoeveel tijd het me ongeveer gaat kosten

Als speler wil ik dat de quiz die ik speel er mooi uitziet, zodat ik gemotiveerd ben om door te spelen

  • De beschrijving van de quiz is in het Nederlands geschreven
  • De speler weet na het lezen van de beschrijving precies hoe de quiz werkt
  • De beschrijving van de quiz is in begrijpelijke taal opgemaakt
  • De informatie over het spelen van de quiz wordt getoond voorafgaand aan het beantwoorden van de vragen
  • De beschrijving van de quiz bevat HTML-layout-elementen, zoals titels, paragrafen en opsommingen
  • Alle elementen op de pagina staan op een logische plek
  • Er zijn kleurcombinaties en plaatjes gebruikt die de pagina verfraaien

Als speler wil ik één vraag tegelijk in mijn scherm zien met de 4 mogelijke antwoorden zodat ik mij op het beantwoorden van één vraag kan concentreren

  • Elke vraag heeft precies 4 antwoordmogelijkheden
  • Er is precies één juist antwoord op de gegeven vraag
  • Er wordt één vraag per keer getoond

Als speler wil ik een antwoord kunnen geven op een vraag zodat ik verder kan naar de volgende vraag

Als speler wil ik weten of mijn gegeven antwoord op een vraag correct is zodat ik kan leren van mijn gegeven antwoorden

  • De gebruiker ziet welk antwoord hij/zij gekozen heeft
  • Het is voor de speler mogelijk precies één antwoord te selecteren
  • De quiz registreert of het gegeven antwoord juist of onjuist is
  • Als de speler het juiste antwoord heeft gegeven moet de speler positieve feedback krijgen
  • Als de speler een onjuist antwoord geeft wordt de speler gewezen op de fout
  • Het juiste antwoord wordt getoond

Als speler wil ik na het beantwoorden van een vraag verder gaan naar de volgende vraag, zodat ik progressie maak in de quiz

  • Na het lezen van de feedback heeft de speler de mogelijkheid naar de volgende vraag te gaan of wordt de speler daar automatisch heen geleid
  • Bij het tonen van een nieuwe vraag zijn de voorgaande vraag en antwoorden niet meer zichtbaar

Als speler wil ik weten hoeveel van de reeds beantwoorde vragen correct zijn beantwoord zodat ik weet wat mijn (voorlopige) score is

  • Bij elke goed beantwoorde vraag wordt de score met 1 verhoogd
  • De actuele score is te allen tijde zichtbaar voor de speler

Als speler wil ik weten wanneer ik de laatste vraag heb beantwoord, zodat ik weet dat de quiz afgelopen is
  • Als de speler de laatste vraag heeft beantwoord worden er geen vragen meer getoond
  • Er wordt een bericht getoond aan de speler als de quiz afgelopen is
